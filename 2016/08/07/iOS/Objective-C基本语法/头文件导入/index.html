头文件导入(include,import异同)

发表于 2016-08-07

include

  • 作用:include是将右边的文件拷贝到当前的位置

  • 注意:

    • include后面不一定要写文件名称 也可以写路径(相对路径/ 全路径)

    • include指令不一定要写在文件的头部

  • 缺点：可能出现重复导入头文件的情况，还需要使用条件编译来解决 .

import

  • 作用

    （1）import的功能和include一样, 是将右边的文件拷贝到当前import的位置

    （2）import可以防止重复导入, 避免程序员去书写头文件卫士

  • 优点: 会自动防止重复拷贝

总结

  • 相同点：

    • 都是预处理指令
    • 都是用来导入头文件，将后面的文件拷贝到当前指令的位置

    • 后面可以跟 <> 和 “” ，使用 <> 和 “”的区别如下

      （1）使用<>

 代表会先从开发工具的编译环境中去查找
/Applications/Xcode.app/Contents/Developer/Platforms/MacOSX.platform/Developer/SDKs/MacOSX10.10.sdk/usr/include/；
 如果开发工具的编译环境中没有找到, 那么会再去系统的编译环境中找/usr/include/stdio.h

（2）使用""
代表会先从当前文件所在的文件夹下面查找；
如果在当前文件所在的文件夹下面没有找到, 那么就回去开发工具的编译环境中去查找；
如果编译环境中没有找到, 那么会再去系统的编译环境中找
  • 不同点：
    • include 可能出现重复导入头文件的情况，解决重复导入还需要条件编译指令来解决
    • import可以防止重复导入, 避免程序员去书写头文件卫士
